# MVP

`mvp/` contiene productos de demostracion construidos sobre `core/templates` y `core/packages`.

## Objetivo

- Validar rapidamente flujos funcionales de negocio.
- Probar integracion real entre front, back e infraestructura local.

## Subproyectos actuales

- `condition-mvp/`: MVP funcional basado en recurso FHIR `Condition`.
- `producto1/`: producto en evolucion con front/back desacoplados.

## Relacion con core

- Se reutilizan contratos, design system, utilidades FHIR y middleware compartido.
